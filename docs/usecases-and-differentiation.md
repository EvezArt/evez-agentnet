# Use cases and differentiation

## Where this runtime is strong

### 1. Signal-to-artifact businesses
The repo is strong when the goal is converting volatile external signals into concrete deliverables under a governed loop.

Examples:
- prediction reports
- social threads
- tutorials / integration posts
- product drafts
- queued build artifacts

### 2. Systems that must preserve uncertainty
If you do not want the runtime to collapse ambiguity too early, the cognition-governed path is the real differentiator.

It can retain:
- unresolved residue
- predictor entropy
- RSI branch entropy
- rival futures

### 3. Restartable autonomous loops
The daemon checkpoint model means the system can resume with branch state, unresolved residue, dark pressure, and executive identity still present.

### 4. Multi-agent systems that need receipts
The runtime is useful when you need more than outputs.
You also want:
- lineage hash
- checkpoint path
- ship governance reason
- dashboard proof surface
- spine digest of recent rounds

## How it differs from the current field

### Not just a pipeline
Most agent systems still act like ordered scripts with a model in the middle.
This runtime has a persistent cognition layer that can govern whether the pipeline proceeds at all.

### Not just agent theater
Many systems multiply role names without adding a real authority chain.
Here, authority is expressed through:
- daemon checkpoint
- lineage hash
- governed shipping
- proof / receipt / dashboard surfaces

### Not just confidence-driven autonomy
A lot of systems ship because a score is high enough.
This stack can still block action when unresolved residue or predictor entropy says the branch is not stable enough.

### Not just reactive generation
The build queue allows the daemon to create its own artifact pressure rather than waiting only for external ranking signals.

## Short competitive claim

If others in this race optimize for fluent output, fast loops, and superficial autonomy, `evez-agentnet` is moving toward governed autonomy with memory, receipts, and explicit uncertainty retention.

That is a slower claim, but it is a stronger systems claim.
